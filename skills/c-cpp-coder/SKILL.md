# C/C++ 代码编写助手

## 注释风格
* 使用 **中文描述 + 英文术语**：遇到术语时使用英文，一般表述使用中文。
* 每个函数、类和方法都应有注释，包含：功能描述、参数说明、返回值说明、异常/边界情况说明。


## 默认命名风格
- 文件名：`snake_case`
- 类型名(类/结构体/枚举类型/using 别名/)：`PascalCase`
- 函数(函数，成员函数，全局函数): `PascalCase`，函数名通常为命令性动词
- 变量(普通变量，全局变量，参数变量)：`snake_case`
- 类成员变量: `snake_case_`（小写 + 下划线 + 结尾下划线）
- 结构体/联合体成员变量: `snake_case`
- 常量: k + `PascalCase`
- 枚举成员: `kPascalCase` 或 `UPPER_SNAKE_CASE`
- 宏: 全大写 + 下划线
- 命名空间: 全小写
- 模板参数: 类型为`PascalCase`, 非类型为小写+下划线


## 代码推荐写法
### 1. 减少代码嵌套
```cpp
void bad_func() {
  if (condition1) {
    if (condition2) {
      if (condition3) {
        // 逻辑
      }
    }
  }
}

void good_func() {
  if (!condition1) 
    return;
  if (!condition2) 
    return;
  if (!condition3) 
    return;
  // 逻辑
}
```

### 2. 无限循环代码
* 使用`for(;;)`而不使用`while(1)`


## 前置条件
1. 必须确定 C++ 规范版本和缩进规则再继续，默认使用 **4 空格缩进**，但需用户确认。
2. 对于 C++17 及以上版本，优先使用 `constexpr` 而非 `#define`，可读性更好。
3. 对于 C++20 及以上版本，优先使用 `std::source_location` 而非 `__FILE__` 和 `__LINE__`。
4. 对于嵌入式开发，默认不使用异常和RTTI, 使用整数类型并减少浮点类型的使用。


## 执行流程
1. **读取项目工程配置**：检查项目根目录的 `CMakeLists.txt` 或 `Makefile`，了解项目结构、依赖库、编译配置、缩进配置，确认 C++ 规范版本。
2. **读取代码风格**：检查项目是否存在 `.clang-format`，了解代码风格要求。
3. **了解项目实现**：读取项目根目录 `README.md`，了解项目实现和功能模块，确保新代码与现有风格和架构一致。
4. **需求分析**：根据用户需求和项目实现，分析需要读取/修改/新建的文件，以及需要修改/实现的功能。
5. **列出待办清单**：根据需求分类，列出待办清单，并严格遵循清单编写代码。
6. **执行代码检查**：代码编写完成后，若 `cpp-lint` skill 存在，使用其检查代码质量和风格，确保符合项目要求。
