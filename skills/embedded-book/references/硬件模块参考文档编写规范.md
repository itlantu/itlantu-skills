# 硬件模块参考文档编写规范

本文档定义了嵌入式硬件模块参考文档（`.md`）的结构、章节划分与编写标准，确保所有模块文档风格统一、内容完整、便于检索。

---

## 1. 文件组织

### 1.1 目录结构

```
references/
├── 模块名称/
│   └── 模块名称.md
```

- 每个模块占用一个独立子目录，目录名使用**英文型号**（如 `MG996R`）。
- 文档主文件与目录同名，便于 `技能` 通过路径直读。
- 若有多张接线图或照片，统一放在该目录下，在 `.md` 中相对引用。

### 1.2 文件命名

- **文件名**：`<型号>.md`（如 `MG996R.md`、`MPU6050.md`）。
- 标题使用中文描述：`# <型号> <中文名称>`（如 `# MG996R 金属舵机 180°`）。

## 2. 章节结构

以下章节按模块类型分为**通用必要章节**和**按类型扩展章节**。编写时应保留通用章节，并根据模块类型选择对应的扩展章节。

### 2.1 通用必要章节（所有模块必须包含）

#### 2.1.1 基本参数

以表格列出模块的核心规格参数，包含但不限于：

| 参数类型 | 示例字段 |
|----------|----------|
| 工作电压 | 范围 + 推荐值 |
| 工作电流 | 典型值 + 峰值/最大值 |
| 通信接口 | I2C / SPI / UART / PWM / 模拟量 / GPIO |
| 工作温度 | 范围 |
| 物理尺寸 | 长 × 宽 × 高（或封装类型） |
| 关键性能指标 | 分辨率 / 精度 / 量程 / 速率 等 |

> 表格格式：`| 参数 | 值 |`，值中可用 **加粗** 标注关键数值。

#### 2.1.2 GPIO 能力需求

| 需求项 | 说明 |
|--------|------|
| GPIO 类型 | 数字输入/输出、PWM、ADC、中断等 |
| 逻辑电平 | 3.3V / 5V / 兼容 |
| 引脚数量 | 总计及各功能引脚数 |
| 外设资源 | 定时器、SPI/I2C/UART 控制器等 |
| 特殊需求 | 上下拉、开漏、中断触发边沿等 |

> 如模块需要电平转换，在此明确说明。

#### 2.1.3 电源需求

| 需求项 | 说明 |
|--------|------|
| 供电电压 | 标称值及容许范围 |
| 供电电流 | 典型值与峰值 |
| 供电方式 | MCU 引脚供电 / 独立电源 / 板载 LDO |
| 共地要求 | 是否需要与 MCU 共地 |
| 去耦建议 | 建议的滤波电容值 |

> 必须注明是否可从 MCU 引脚直接供电，以及供电不足的典型后果。

#### 2.1.4 接线定义

以表格列出模块每根引脚/线序的定义：

| 序号 | 引脚名 / 线色 | 功能 | 连接目标 | 备注 |
|------|--------------|------|----------|------|
| 1 | VCC (红) | 电源正极 | 外部 5V / MCU 5V | — |

- 物理接口（排针 / 排母 / 端子座 / 连接器型号）也应在表前或表后简要说明。
- 对于芯片类模块（如传感器 breakout 板），列出 I2C/SPI 引脚编号及 MCU 对应连接。

#### 2.1.5 驱动要点

以编号列表给出关键的驱动注意事项，每条应涵盖一个独立要点。推荐的要点类别：

1. **上电初始化**：复位时序、初始状态设置。
2. **数据读写**：轮询 vs 中断、阻塞 vs DMA。
3. **异常处理**：超时、校验失败、通信重试。
4. **低功耗**：休眠/唤醒策略。
5. **安全边界**：电压/电流/温度保护。
6. **多实例**：同一总线挂多个设备的注意事项。

#### 2.1.6 常见故障排查

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| ... | ... | ... |

- 每个条目应可独立闭环，即按"现象→原因→解决"即可定位修复。
- 覆盖通信、供电、数据异常、物理连接四类典型问题。

### 2.2 按模块类型扩展章节

#### 2.2.1 执行器（舵机、电机驱动、步进电机等）

**PWM / 控制信号规范**

| 参数 | 值 |
|------|-----|
| 信号类型 | PWM / 脉冲 / 模拟电压 / 数字指令 |
| 频率 | xx Hz |
| 周期 | xx ms |
| 有效脉宽范围 | xx ~ xx ms |
| 信号逻辑电平 | 3.3V / 5V |

**控制量换算公式**

给出输入量（角度、速度、步数等）与输出量（脉宽、CCR、频率等）之间的数学关系，使用 LaTeX 公式。

**控制量对照表**

列出关键输入-输出对应值，便于快速查表。

**定时器 PWM 配置（平台通用）**

| 参数 | 要求 | 说明 |
|------|------|------|
| PWM 频率 | **xx Hz** | 周期 xx ms |
| 定时器时钟源 | ≥ x MHz | — |
| ARR | xxxx | — |
| CCR 范围 | xx ~ xx | 对应有效脉宽范围 |

> 给出高/低分辨率两套方案，并注明公式：`f_PWM = f_CLK / ((PSC+1) × (ARR+1))`。

#### 2.2.2 传感器（温度、湿度、IMU、光照、距离等）

**通信协议**

| 参数 | 值 |
|------|-----|
| 协议类型 | I2C / SPI / UART / OneWire / 模拟量 |
| 从机地址 | 7-bit / 8-bit 地址（I2C） |
| 时钟速率 | 标准/快速/高速模式 |
| 数据格式 | 大端 / 小端，有/无符号 |

**数据读取流程**

以步骤列表或流程图说明初始化→触发测量→等待→读取的完整时序。

**寄存器 / 指令说明**

| 寄存器地址 | 名称 | 读写 | 位宽 | 描述 |
|------------|------|------|------|------|
| 0x00 | WHO_AM_I | R | 8 | 器件 ID |

- 关键配置寄存器必须列出。
- 数据寄存器需说明位拼接/符号扩展方式。

**数据解析与换算**

给出从原始字节到物理量的换算公式（如 `温度 = raw × 0.01`）。

**校准参数（如有）**

说明出厂校准值存储地址或用户校准流程。

#### 2.2.3 显示模块（LCD、OLED、TFT、LED 矩阵等）

**通信协议**

| 参数 | 值 |

| 参数 | 值 |

**显存布局与寻址**

说明页/列/行地址设置方式，以及像素-字节映射关系。

**字体与绘图（可选）**

## 3. 格式规范

### 3.1 标题层级

- `#` 模块标题（仅一个）。
- `##` 主章节标题（编号 + 名称，如 `## 1. 基本参数`）。
- `###` 子章节标题（如 `### 高分辨率方案`）。
- 同级标题之间用 `---` 分隔（可选，推荐在章节末尾使用）。

### 3.2 表格

- 所有参数表统一使用 `| 参数 | 值 |` 或 `| 需求项 | 说明 |` 表头。
- 数值统一标注单位（如 `4.8V ~ 6.0V`），推荐值用 **加粗**。
- 长文本说明在表格外以 `>` 引用块补充。

### 3.3 公式

- 简单公式用行内 LaTeX（`$...$`）。
- 复杂公式用块级 LaTeX（`$$...$$`）。
- 涉及平台相关的参数应给出通用公式而非绑定特定平台代码。

### 3.4 代码

- 不包含平台特定代码（如 `ledcWrite()`、`HAL_TIM_PWM_Start()`）。
- 伪代码或通用配置逻辑允许以平台无关方式描述。
- 确有需要的初始化命令序列用十六进制字节列表表示。

### 3.5 引用与提示

- 使用 `>` 引用块标注重要提示、平台差异说明、注意事项。
- 格式：
  ```
  > **提示类型**：具体内容。
  ```

## 4. 编写清单

创建新模块文档时，按以下清单逐项检查：

- [ ] 文件名与目录名使用英文型号
- [ ] 第 1 行标题为 `<型号> <中文名称>`
- [ ] 基本参数表已填写
- [ ] GPIO 能力需求表已填写
- [ ] 电源需求表已填写（含供电方式与共地要求）
- [ ] 接线定义表已填写（含线色/引脚名+功能+连接目标）
- [ ] 驱动要点已列出（≥ 5 条）
- [ ] 常见故障排查表已填写（≥ 4 条）
- [ ] 按模块类型选择扩展章节并填写
- [ ] 控制量换算公式使用 LaTeX（如适用）
- [ ] 无平台特定代码
- [ ] 已收录模块索引表在 `SKILL.md` 中更新
